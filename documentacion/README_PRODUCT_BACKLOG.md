# Product Backlog

Este documento contiene el Product Backlog del proyecto, donde se listan todas las funcionalidades, historias de usuario y tareas pendientes priorizadas para el desarrollo.

## Historias de Usuario y Funcionalidades
- Como usuario, quiero registrarme y autenticarme para acceder a mis recursos.
- Como usuario, quiero crear, editar y eliminar publicaciones.
- Como usuario, quiero organizar mis publicaciones en carpetas y bibliotecas.
- Como usuario, quiero gestionar mi diario personal.
- Como usuario, quiero editar mi perfil y eliminar mi cuenta.
- Como usuario, quiero ver publicaciones públicas de otros usuarios.
- Como usuario, quiero una interfaz accesible y usable.
- Como administrador, quiero gestionar usuarios y contenidos.

## Priorización
Las historias están priorizadas según el valor para el usuario y la complejidad técnica. Se revisan y actualizan en cada Sprint.

## Actualización
El Product Backlog se actualiza en cada reunión de planificación de Sprint y se encuentra reflejado en los archivos de cada Sprint.
